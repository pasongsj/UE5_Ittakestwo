

---------------------------------지우면 안되는 애들---------------------------------
.uproject 파일 - 언리얼 에디터 구동 시 해당 프로젝트 정보를 포함하는 JSON 파일.

Config 폴더 - 프로젝트의 언리얼 환경설정 파일을 담고있는 폴더
	   - 지우면 설정값을 없어지고 언리얼 엔진에서 제공하는 기본세팅으로 적용
	   - 지우고 빌드 다시해도 Config폴더가 다시 안생김  

Content 폴더 - 에디터에서 사용할 에셋들 담는 폴더

Plugins 폴더 - 외부 thirdparty 플러그인을 사용하는 경우 직접 생성
	    - 엔진에서 제공해주는 플로그인은 여기에 생성 안됨

Source 폴더 - C++프로젝트 전용. 모듈 소스, 모듈 파이프라인, C#소스파일 들어있음
--------------------------------------------------------------------------------------



---------------------------------지워도 문제는 안됨---------------------------------
.sln 파일 - 생략

.vs 폴더 - Generate 과정에서 생성되는 폴더(숨김 파일임)
           - 빌드 빠르게 하려고 초기화 및 구조를 기록하는 폴더

Binaries 폴더 - 프로젝트에 대한 에디터 전용 DLL 파일이 담긴 폴더
	     - 이거 지우면 유동민이 겪었던 그 오류가 당신들을 맞이할 것임.
	     - 대신 yes누르면 다시 생성됨

Build 폴더 - 언리얼 프로젝트 패키징하면 생성됨

Intermediate 폴더 - 소스코드 및 빌드 과정을 거치면 생성됨
 	           - 게임 디렉터리에서 셰이더가 여기 저장됨
--------------------------------------------------------------------------------------



---------------------------------지워도 되는데 딱히 이득은 없음---------------------------------

Saved 폴더 - 자동 저장, 스크린샷, 빌드 파일, 백업 임시파일, 로그 등 여기 저장됨
	   - 프로젝트 진행 하다보면 용량이 커져서 Config파일 빼고 백업해두면 용량을 크게 줄일 수 있음
